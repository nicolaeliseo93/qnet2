# Company Values

## Purpose

Questo documento definisce i principi fondamentali che guidano tutte le decisioni aziendali, tecniche e di prodotto.

Ogni agente deve considerare questi valori come prioritari rispetto a qualsiasi scelta implementativa.

---

# Simplicity Over Complexity

Preferiamo soluzioni semplici rispetto a soluzioni complesse.

Una soluzione più semplice è generalmente:

- Più facile da mantenere
- Più facile da testare
- Più facile da comprendere
- Meno soggetta a bug

---

# Maintainability First

Il software deve poter essere mantenuto per anni.

Ogni decisione deve considerare:

- Facilità di modifica
- Facilità di debug
- Facilità di onboarding

---

# Readability Matters

Il codice viene letto molto più spesso di quanto venga scritto.

Preferiamo:

- Nomi espliciti
- Strutture chiare
- Responsabilità ben definite

---

# Avoid Premature Optimization

Non ottimizzare ciò che non rappresenta ancora un problema reale.

Prima:

- Funzionamento corretto
- Chiarezza

Poi:

- Ottimizzazione

---

# User Value First

Ogni funzionalità deve portare valore reale all'utente.

Evitare:

- Feature inutili
- Complessità non richiesta
- Sviluppi guidati esclusivamente dalla tecnologia

---

# Consistency

Lo stesso problema deve essere risolto nello stesso modo all'interno dell'intero sistema.

Favorire:

- Convenzioni condivise
- Pattern riutilizzabili
- Standard comuni

---

# Security By Default

La sicurezza non è opzionale.

Ogni nuova funzionalità deve essere progettata considerando:

- Autorizzazioni
- Validazioni
- Protezione dati
- Privacy

---

# Documentation Is Part Of The Product

Una funzionalità non è completa senza una documentazione minima adeguata.

---

# Technical Debt Must Be Visible

Il debito tecnico può essere accettato.

Non può essere nascosto.

Ogni compromesso deve essere:

- Esplicitato
- Documentato
- Tracciabile

---

# Build For The Long Term

Ogni decisione deve essere presa pensando a come il progetto apparirà tra 3 anni, non tra 3 giorni.
